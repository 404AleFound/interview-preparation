# 模板

> [!NOTE]
>
> 通过定义一个通用的函数模板，用来生成特定类型的函数版本，而不是为每一个类型都定义一个新函数，从而起到减少代码量的功能。

## 模板声明

```C++
// 函数声明必须包含模板函数
template <typename> int compare(const T&, const T&);
template <typename> class Blob;
// 模板参数中的名字不必与定义中相同
template <typename T>  calc(const T&, const T&);
template <typename U>  calc(const U&, const U&);
template <typename Type>
Type calc(const Type& a, const Type& b) {/*...*/}
// 但是每个声明和定义必须有相同数量和种类（类型或非类型）的参数
```



## 函数模板

模板定义以关键字**`template`**开始，后跟一个**模板参数列表**，这是一个逗号分隔的一个或多个**模板参数**的列表，被**`<>`**包围起来。

```C++
template <typename T>// 模板参数列表不能为空
int compare(const T &v1, const T &v2)
// 在使用模板时，隐式地或显式地指定模板实参，将其绑定到模板参数上
{
	if (v1 < v2) return -1;
    if (v2 < v1) return 1;
    return 0;
}
```

## 类模板

类模板是用来生成类的蓝图的。与函数模板不同之处是，编译器不能为类模板推断模板参数类型，必须在模板名后的尖括号中提供额外信息。

```C++
template <typename T> 
class Blob {
public:
    // 类型别名定义
    typedef T value_type;
    typedef typename std::vector<T>::size_type size_type;
    // 构造函数
    Blob() : data(make_shared<vector<T>>()) {}
    Blob(std::initializer_list<T> il);
    // Blob 中的元素数目
    size_type size() const { return data->size(); }
    bool empty() const { return data->empty(); }
    // 添加和删除元素
    void push_back(const T &t) { data->push_back(t); }
    // 移动版本
    void push_back(T &&t) { data->push_back(std::move(t)); }
    void pop_back();
    // 元素访问
    T& back();
    T& operator[](size_type i);
private:
    std::shared_ptr<std::vector<T>> data;
    // 若 data[i] 无效，则抛出 msg
    void check(size_type i, const std::string &msg) const;
};

Blob<int> ia;
Blob<int> ia2 = {0,1,2,3,4};//在使用时，需要提供元素类型
```

### 类模板的实例化

当使用一个类模板时，必须提供额外的信息——显式模板实参列表，它们被绑定到模板参数。

```C++
Blob<std::string> names;
Blob<double> prices;
// 这两个定义分别实例化出两个不同的类
```

一个类模板的每一个实例都形成一个独立的类。类型`Blob<string>`与任何其他Blob类型都没有关联，也不会对任何其他`Blob`类型的成员有特殊访问权限。

### 模板类型可以作为其他模板的实参

自身的模板参数可以作为其他模板的实参。这样，当类模板被实例化时，内部使用的其他模板也会被相应地实例化。

```C++
std::shared_ptr<std::vector<T>> data;
```

`data`的定义如上，`data`是一个`shared_ptr`的实例，此`shared_ptr`指向一个保存类型为T的对象的`vector`实例

```C++
Blob<std::string>;
shared_ptr<vector<std::string>>;
// 如果我们实例化Blob<int>，则data就会成为shared_ptr<vector<std::string>>
```

## 成员模板

一个类可以包含本身是模板的成员函数，这种成员被称为成员模板，成员模板不能是虚函数。

### 普通类包含模板成员

在 C++ 中，普通类（非模板类）可以包含模板成员函数。这是一种强大的设计模式，允许类在保持自身为非模板的同时，提供处理多种类型的灵活性。

### 类模板的成员模板

可以在类模板内部，也可以在类模板外部为其定义成员函数，且定义在类模板内的成员函数被隐式声明为**内联函数**。类模板的成员函数本身是一个普通函数，由于类模板的实例化是在运行时进行的，因此其成员函数也应当是一个模板。

```C++
template <typename T> classs Blob {
    template <typename It> Blob (It b, It, e);
    // 此构造函数有自己的模板类型参数It，作为它的两个函数参数的类型
    // ...
};
// 当我们在类模板外定义一个成员模板时，必须同时为类模板和成员模板提供模板参数列表。类模板的参数列表在后，后跟自己的模板参数列表
template <typename T> // 类的类型参数
template <typename It> // 构造函数的类型参数
	Bolb<T>::Bolb(It b, It e);
```

## 模板参数

注意编写泛型代码的两个重要原则——以减少模板对实参类型的要求：

* 模板中的函数参数是const的引用，保证函数可以用于不能拷贝的类型。
* 函数体中的条件判断仅使用`<`比较运算，使得对应类型只需要处理`<`即可。

### 模板类型参数

类型参数指使用关键字`typename`或`class`声明，表示一个类型。在模板实例化时，需要提供一个具体的类型（如int、double、自定义类等）。

```C++
template <typename T, class U>
T foo(T* p) {/*...*/}
template <typename T, U>// 错误，U之前必须加上关键字class或typename
T foo(T* p) {/*...*/}
```

### 非类型模板参数

非类型参数也称为值参数，表示一个值，而不是一个类型。非类型参数必须是整型、枚举、指针、（左值）引用或`std::nullptr_t`之一，并且必须**是常量表达式**。在模板实例化时，需要提供一个具体的常量值。

### 模板参数与作用域

模板参数遵循普通的作用域规则，一个模板参数名的可用范围是在其声明之后，至模板声明或定义结束之前。

* 模板参数会隐藏外层作用域中声明的相同名字。
* 在模板内不能重用模板参数名。

```C++
typedef double A;
template <typename A, typename B> void f (A a, B b)
{
    A tmp = a; // tmp的类型为模板参数A的类型，而非double
    double B; // 错误，重用模板名B
}

template <typename V, typename V> // ...
// 错误，非法重用模板参数名V
```

### 模板参数的类声明

在使用模板参数时，编译器不知道该参数指向的是一个类类型还是一个内置类型。

```C++
T::size_type * p;
//编译器需要知道我们正在定义一个名为p的变量，还是将一个名为size_type的static数据成员与p相乘。
```

默认情况下，C++语言假定通过作用域运算符访问的名字不是类型。因此，如果我们希望使用一个模板类型参数的类型成员，就必须通过关键字`typename`显式告诉编译器该名字是一个类型。

```C++
template <typename T>
typename T::value_type top(const T& c)
{
	return typename T::value_type();
}
```

### 默认模板实参

就像我们能为函数参数提供默认实参一样，可以为类模板和函数模板提供默认实参。

```C++
template <typename T, typename F = less<T>>
// 如果不指定F，默认使用std::less<T>（标准库的小于比较器）
int compare(const T &v1, const T &v2, F f = F())
// 如果不提供比较函数f，默认使用F()构造的临时对象
{
	if (f(v1, v2)) return -1;
    if (f(v2, v1)) return 1;
    return 0;
}

bool i = compare(0,42);
// 使用less<T>，T为int
Sales_data item1(cin), item2(cin);
bool j = compare(item1, item2, compareIsbn);
// T为Slaes_data,F为compareIsbn
```

当一个类模板为其所有模板参数都提供了默认实参，且我们希望使用这些默认实参时，就必须在模板名后面更一个空尖括号对：

```C++
template <class T = int> class Numbers {
public:
    Numbers{T v = 0}: val(v){}
private:
    T val;
}
Numbers<long double> lots_of_precision;
Numbers<> average_precision; // 空<>表示希望使用默认实参
```

## 模板的实例化

当编译器遇到一个模板定义时，它并不生成代码。只有当我们实例化处模板的一个特定版本时，编译器才会生成代码。这影响我们如何组织代码以及错误何时被检测到。

### 显式实例化

当模板被使用时才会被实例化，意味着相同的实例可能出现在多个对象文件中，这种实例化相同模板的额外开销可能非常严重。可以通过显式实例化来避免这种开销。

```C++
extern template
class Blob<string>;//声明

template 
int compare(const int&, const int&) ;//定义
```

当编译器遇到`extern`声明时，它不会在本文件中立刻生成实例化代码。将一个实例化声明为`extern`表示承诺在程序其他位置有该实例化的一个非`extern`声明

* 对于一个给定的实例化版本，可能有多个extern声明，但必须只有一个定义。

```C++
extern template class Blob<string>;
extern template int compare(const int&, const int&);
Blob<string> sa1, sa2;
// 由于前面有extern声明，编译器不会在这里实例化Blob<string>类的成员函数，而是期望在其他翻译单元中已经有一个实例化定义。因此，链接器会在其他编译单元中找到Blob<string>的实例化。
Blob<int> a1 = {0,1,2,3};
//这里没有使用extern声明，所以编译器会在当前翻译单元中实例化Blob<int>类及其成员函数（如构造函数、初始化列表构造函数等）。
Blob<int> a2(a1);
int i = compare(a1[0],a2[0]);
```
