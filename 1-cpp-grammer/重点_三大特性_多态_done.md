# 三大特性——多态

> [!NOTE]
>
> **三大特性——多态**
>
> * 动态绑定
>   * 动态绑定也称运行时绑定，指程序运行期间根据对象的实际类型来确定调用哪个函数。
>   * 实现机制：通过基类的引用（或指针）调用一个虚函数时将发生动态绑定。
> * 类型转换
>   * 静态类型和动态类型：针对引用或指针而言，引用或指针的类型为静态类型，其指向对象的类型为动态类型。
>   * 不能将一个派生类指针绑定在基类指针
>   * 在派生类对象和基类对象之间不存在类型转换
>   * 派生类向基类转换的可访问性

## 动态绑定

### 什么是动态绑定：运行时函数绑定

动态绑定，也称为运行时绑定（Run-time Binding），是指在程序运行期间根据对象的实际类型来确定**调用哪个函数**。动态绑定与静态绑定（编译时绑定）相对。

- 静态绑定：在编译时确定函数调用，包括普通函数调用、模板函数、重载函数等。
- 动态绑定：在运行时确定函数调用，通过虚函数机制。

在C++语言中，当我们使用基类的引用（或指针）调用一个虚函数时将发生动态绑定。

### 如何实现动态绑定：虚函数表

每个包含**虚函数**的类（或有虚函数的类的派生类）都有一个虚函数表。虚函数表是一个函数指针数组，其中存放着该类所有虚函数的地址。每个对象中都有一个隐藏的指针`vptr`指向该类的虚函数表。

当通过基类指针或引用调用虚函数时，编译器会生成代码来通过对象的`vptr`找到虚函数表，并在表中查找相应函数的地址，然后调用该函数。

**动态绑定的条件：**

* 基类中声明虚函数（使用`virtual`关键字）。
* 派生类中覆盖（重写）基类的虚函数（函数签名相同）。
* 通过基类的指针或引用来调用虚函数。

## 类型转换

类型转换来自于类的继承机制，一个派生类对象包含多个组成部分：

* 属于派生类对象自己的部分
* 属于继承的基类的部分。若有多个基类，则对应的基类部分也有多个。

正是基于派生类对象中包含有与其基类对应的组成部分，我们能把派生类的对象当成基类对象使用，即可以将基类的指针或引用绑定到派生类对象中的基类部分上。

### 静态类型和动态类型

由于派生类和基类存在类型转换，且该转换是在运行时确定的，故引入**静态类型和动态类型**概念。

* 动态类型是表达式指向内存中对象的类型，表达式的静态类型是其在声明时的类型。
* 当表达式既不是引用也不是指针时，其静态类型和动态类型是一致的。

```C++
Derived d;
Base* pb = &d; // 静态类型：Base*，动态类型：指向 Derived 对象
pb->who(); // 输出：I am Derived （多态体现）

Base& rb = d; // 静态类型：Base&，动态类型：Derived
rb.who(); // 输出：I am Derived

Base b2 = d; // 静态类型和动态类型均为 Base （对象切片发生）
b2.who(); // 输出：I am Base
```

### 注意：不能将一个派生类指针绑定在基类指针

* 派生类对象包含一个基类部分，所以基类的指针或引用可以指向该派生类对象的基类部分。
* 而基类对象可以独立存在，因此不能将一个派生类指针绑定在基类指针上。

```C++
class Base {
public:
    void foo() {}
};

class Derived : public Base {
public:
    void bar() {}
};

Derived d;
Base* pb = &d;   
// 正确，基类指针指向派生类对象的基类部分
Base b;
// Derived* pd = &b;
// 错误，基类对象不能赋给派生类指针（不能隐式转换）

```

### 注意：在派生类对象和基类对象之间不存在类型转换

* 派生类向基类的自动转换只对指针或应用类型有效。
* 当使用一个派生类对象为一个基类对象初始化或赋值时，只有该派生类对象中的基类部分会被拷贝、移动或赋值，它的派生类部分将被切掉（sliced down

```C++
class Base {
public:
    int baseData = 1;
    virtual void print() 
    { std::cout << "Base: " << baseData << std::endl; }
};

class Derived : public Base {
public:
    int derivedData = 100;
    void print() override 
    { std::cout << "Derived: " << baseData << ", " << derivedData << std::endl; }
};

Derived d; d.baseData = 2; d.derivedData = 200;

Base b = d;  // 对象切片：只拷贝 Base 部分，Derived 部分被“切掉”
b.print();   // 输出 Base: 2 —— 派生类信息丢失

Base& br = d;  // 引用或指针无切片
br.print();    // 输出 Derived: 2, 200 （多态）

Base* bp = &d;
bp->print();   // 输出 Derived: 2, 200 （多态）

```

### 注意：派生类向基类转换的可访问性

| 继承方式 | 用户代码是否能转换 | 派生类成员/友元是否能转换 |
  | -------- | ------------------ | ------------------------- |
  | 公有继承 | 可以               | 可以                      |
  | 保护继承 | 不可以             | 可以                      |
  | 私有继承 | 不可以             | 可以                      |

```C++
class B {};

class D_public : public B {
public:
    void test() {
        B* pb = this;  // OK，公有继承，成员函数内可访问
    }
};
class D_protected : protected B {
public:
    void test() {
        B* pb = this;  // OK，成员函数可以访问protected继承的基类
    }
};
class D_private : private B {
public:
    void test() {
        B* pb = this;  // OK，成员函数可以访问private继承的基类
    }
};
// =======================================
D_public dp;
B* pb1 = &dp;
// OK，用户代码可访问公有继承的基类部分
// =======================================
D_protected dprot;
// B* pb2 = &dprot; 
// 编译错误，用户代码不可访问protected继承的基类部分
// =======================================
D_private dpriv;
// B* pb3 = &dpriv; 
// 编译错误，用户代码不可访问private继承的基类部分
// =======================================
```
