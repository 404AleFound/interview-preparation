# 三大特性——封装

> [!NOTE]
>
> **三大特性——封装**
>
> * 封装的意义：封装（Encapsulation）是面向对象编程中的核心特性之一，指的是**将数据（成员变量）和操作数据的代码（成员函数）绑定在一起，隐藏对象的内部实现细节，只对外暴露有限的接口**。
>
> * 类的基本语法和概念
>
> * 类的构造拷贝控制
>
> * 类的控制访问：实现封装的关键技术
>
>   * `private`：允许类内和友元访问
>   * `public`：彻底开放访问权限
>   * `protected`：允许类内、友元、派生类访问
>   * 类的友元：友元函数、友元类、友元成员函数
>
>   类的作用域：名字查找
>
> 

## 类的基本概念

```C++
class class_name{
    // 默认为private
access_specifiers:
    return_type func1(param);
    value_type mem_name;
}

struct class_name{
    // 默认为public
access_specifiers:
    return_type func1(param);
    value_type mem_name;
}
```

## 类的构造拷贝控制

### 构造函数

每一个类都定义了它对象被初始化的方式，类通过一个或几个特殊的成员函数来控制其对象的初始化过程，这些函数被叫做构造函数。构造函数的任务是初始化类对象的数据成员，无论何时只要类的对象被创建，就会执行构造函数。

```C++
class class_name{
public:
    class_name(params);// 构造函数
    class_name();// 默认构造函数
}
```

* 注意，构造函数的名字与类类型的名字相同
* 构造函数不返回任何值
* 一个类种可以有多个构造函数，不同的构造函数必须在参数数量或者参数类型上有所区别。
* 若一个构造函数不接受任何参数，则称其为默认构造函数。

### 合成的默认构造函数

类通过默认构造函数来控制默认初始化的过程。由于其的重要程度，如果类没有显式地定义构造函数，那么编译器就会为我们隐式地定义一个默认构造函数。

合成的默认构造函数规则如下：

* 如果存在类内的初始值，用它来初始化成员。
* 否则，默认初始化该成员。

```C++
class_name() = default;
```

在C++11种，如果我们需要默认的行为，那么可以通过在参数列表后面写上`= default`来要求编译器生成构造函数。如果`=defalut`在类的内部，则默认构造函数是内联的；如果它在类的外部，则该成员默认情况下不是内联的。

### 拷贝构造函数

```C++
class class_name{
public:
    class_name(const class_name & params);// 拷贝构造函数
}
```

### 析构函数

```C++
class class_name{
public:
    ~class_name();
}
```

## 类的控制访问

### 访问权限修饰符

| 访问权限修饰符 | 说明                         | 可访问范围                   |
| -------------- | ---------------------------- | ---------------------------- |
| `private`      | 私有成员，类外不可访问       | 仅类内及友元函数、友元类访问 |
| `protected`    | 保护成员，类内和派生类可访问 | 类内、友元和派生类访问       |
| `public`       | 公共成员，任何地方都可访问   | 任何代码                     |

### 类的友元

**可以声明为友元的对象**

通过友元技术，类可以允许其他类或者函数访问它的非公有成员。

* 类可以把非成员函数定义为友元
* 类可以把其他类定义为友元
* 类可以把其他类的成员函数定义为友元

```C++
void func1(); //单独的类外声明
class class_name_1 {/*...*/};
class class_name_2 {/*...*/};

class class_name{
friend void func1(); // 友元函数
friend class class_name_1; // 友元类
// 将一个类指定为友元，那么其成员函数可以访问此类的所有成员
friend void class_name_2::func2(); // 类的成员函数指定为友元
}
```

* 友元声明只能出现在类定义的内部，但是类内出现的具体位置不限。
* 友元不是类的成员，也不受它所在区域访问控制级别的约束
* 友元声明仅仅指定了访问的权限，如果我们希望类的用户能够调用某个友元函数，那么我们就必须在友元声明外再专门对函数进行一次声明

**友元和函数重载**

如果一个类想把一组重载函数声明成它的友元，它需要对这组函数中的每一个分别声明：

```C++
// 两个重载的 storeOn 函数
extern std::ostream& storeOn(std::ostream &, Screen &);
extern BitMap storeOn(BitMap &, Screen &);
// extern是外部链接的声明，用来告诉编译器：该变量或函数在其他地方定义（通常在另一个源文件中）这里只是声明，不是定义。

class Screen {
    // 将接受std::ostream&的storeOn声明为友元函数，
    // 这样该函数就能访问Screen非公开(private)的成员
    friend std::ostream& storeOn(std::ostream &, Screen &);
    // 但并没有将接受BitMap&参数的另外一个重载版本声明为友元，
    // 所以那个版本不能访问Screen的私有成员
};
```

## 类的作用域

每个类都会定义它自己的作用域。在类的作用域之外，普通的数据和函数成员只能由对下个、引用或者指针使用成员访问运算符来访问。不论哪种情况，跟在运算符之后的名字都必须是对应类的成员。

```C++
Screen::pos ht = 24, wd = 80;
Screen scr(ht, wd, ' ');
Screen *p = &scr;
char c = scr.get();
c = p->get();
```

特别的，对于函数的返回类型，由于其出现在作用域说明符之前，因此如果返回的类型是位于类内的一个成员类型，则在返回值类型前面也应当添加作用域说明符。

```C++
class_name::ScreenIndex
class_name::addScreen(const Screen &s){};
```

### 名字查找

名字查找过程如下：

* 首先，在名字所在的块中寻找其声明语句，只考虑在名字的使用之前出现的声明
* 如果没有找到，继续查找外层的作用域
* 如果最终没有找到匹配的声明，则程序报错

但是对于定义在类内部的成员函数来说，名字查找的方式有所区别：

* 首先，编译成员的声明
* 知道类全部可见后才编译函数体

```C++
typedef double Money;
string bal;
class Account {
public:
    Money balance() {return bal;}
private:
    Money bal;
};
// 当看到balance函数的声明语句时，它将在Account类的范围内寻找对Money的声明，编译器只考虑Accout中在使用Money前出现的声明，因为没找到匹配的成员，编译器会在Account的外层作用域中查找。
// 而函数体中的return bal，在整个类可见后才被处理，因此该函数return饭hi名为bal的成员，而非外层作用域的string对象。
```
