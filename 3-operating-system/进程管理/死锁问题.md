# 死锁问题
## 模拟死锁现象
要使得死锁发生，就要是实现死锁发生的四个条件：
- **互斥条件**：多个线程不能同时使用同一个资源。
- **持有并等待**：当线程 A 已经持有了资源 1，又想申请资源 2，而资源 2 已经被线程 C 持有了，所以线程 A 就会处于等待状态，但是线程 A 在等待资源 2 的同时并不会释放自己已经持有的资源 1。
- **不可剥夺条件**：当线程已经持有了资源 ，在自己使用完之前不能被其他线程获取，线程 B 如果也想使用此资源，则只能在线程 A 使用完并释放后才能获取。
- **环路等待条件**：两个线程获取资源的顺序构成了环形链。


## 实验

### 死锁模拟
如下所示，设计一个模拟的死锁现象。线程 1 先获得互斥锁 m1，之后尝试获取互斥锁 m2；线程 2 先获得互斥锁 m2，之后尝试获取互斥锁 m1，如此便形成了环路等待条件；且两个线程只有在全部执行完成后，才会释放已经获得的互斥锁；且在持有锁的时候，如果发生阻塞便会一直等待，不会放弃当前拥有的锁。
```cpp
#include <iostream>
#include <thread>
#include <mutex>
#include <chrono>
#include <atomic>

static std::mutex m1, m2;

void func1() {
    std::cout << "func1: trying to get m1." << std::endl;
    m1.lock();
    std::cout << "func1: have got m1." << std::endl;
    std::this_thread::sleep_for(std::chrono::seconds(1));
    std::cout << "func1: trying to get m2." << std::endl;
    m2.lock();
    std::cout << "func1: have got m2." << std::endl;
    m2.unlock();
    m1.unlock();
    f1 = true;
    std::cout << "func1: finished." << std::endl;
}

void func2() {
    std::cout << "func2: trying to get m2." << std::endl;
    m2.lock();
    std::cout << "func2: have got m2." << std::endl;
    std::this_thread::sleep_for(std::chrono::seconds(1));
    std::cout << "func2: trying to get m1." << std::endl;
    m1.lock();
    std::cout << "func2: have got m1." << std::endl;
    m1.unlock();
    m2.unlock();
    f2 = true;
    std::cout << "func2: finished." << std::endl;
}

int main()
{
	std::thread t1(func1);
    std::thread t2(func2);
    if (t1.joinable()) t1.join();
    if (t2.joinable()) t2.join();
	return 0;
}
```

### 实验小结
这部分关键学习如何使用 gdb 工具进行简答的死锁检查，相关命令如下：
```bash
# 开启调试编译，添加关键字 -g
g++ -std=c++11 -g -o ./bin/deadlock deadlock.cpp
# 启动 gdb 运行编译好的文件
gdb ./bin/deadlock
# 运行
(gdb) run

```

## 避免死锁的发生
为了避免死锁的方式，方法之一就是破坏环路等待条件，对上述实验代码进行如下改进：
```cpp
// ...
void func1() {
    std::cout << "func1: trying to get m1." << std::endl;
    m1.lock();
    std::cout << "func1: have got m1." << std::endl;

    std::this_thread::sleep_for(std::chrono::seconds(1));

    std::cout << "func1: trying to get m2." << std::endl;
    m2.lock();
    std::cout << "func1: have got m2." << std::endl;

    m2.unlock();
    m1.unlock();

    f1 = true;
    std::cout << "func1: finished." << std::endl;
}

void func2() {
    std::cout << "func2: trying to get m1." << std::endl;
    m1.lock();
    std::cout << "func2: have got m1." << std::endl;

    std::this_thread::sleep_for(std::chrono::seconds(1));

    std::cout << "func2: trying to get m2." << std::endl;
    m2.lock();
    std::cout << "func2: have got m2." << std::endl;

    m2.unlock();
    m1.unlock();

    f2 = true;
    std::cout << "func2: finished." << std::endl;
}
// ...
```
