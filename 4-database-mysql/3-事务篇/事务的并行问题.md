# 事务的并行问题

mysql 是允许多个客户端连接着，这意味着 mysql 中会出现事务的并行情况，那么在同时处理多个事务的时候，就可能出现脏读（dirty read），不可重复读（non-repeatable），幻读（phantom read）问题

## 脏读

### 什么是脏读？
如果一个事务读到另一个尚未提交事务修改的数据，那么就意味着出现了脏读。

## 不可重复读
### 什么是不可重复读？
如果一个事务多次读取同一个数据，如果出现前后两次读到的数据不一样，那么就意味着出现了不可重复读。

## 幻读
### 什么是幻读？
在一个事务内多次查询某个符合条件的记录数量，如果出现前后两次查询到的记录数量不一样的情况，那么就意味着出现了幻读。
